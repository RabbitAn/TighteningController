# TighteningController

基于 WPF 的拧紧控制器客户端，通过 Open Protocol 协议与拧紧控制器（如 Atlas Copco）进行 TCP 通信，实现拧紧过程监控与管理。

## 功能特性

- **连接管理** - 支持 TCP 连接、自动重连（最多 5 次）、心跳保活（10s 间隔 / 15s 超时检测）
- **拧紧结果订阅** - 实时接收拧紧结果（扭矩、角度、OK/NOK 状态），支持 Rev1~Rev15 及 Rev999
- **VIN 码管理** - 发送/取消 VIN 码关联
- **参数集选择** - 选择 PSET（参数集）和 Job
- **批次控制** - 设定批次大小、批次计数、批次清零
- **拧紧控制** - 开始拧紧、使能/禁用工具
- **报警处理** - 报警订阅、实时报警通知、远程确认报警
- **自动/手动模式** - 订阅并显示控制器当前模式
- **数据查询** - 请求工具数据（序列号、紧固次数、校准日期）、查询历史拧紧结果、离线结果计数

## 快速开始

### 环境要求

- Visual Studio 2022 或更高版本
- .NET 9 SDK

### 运行步骤

1. 克隆仓库

```bash
git clone https://github.com/RabbitAn/TighteningController.git
```

2. 用 Visual Studio 打开 `TighteningController.sln`

3. 还原 NuGet 包并编译运行

### 使用说明

1. 输入控制器 IP 地址和端口号（默认 `192.168.5.212:4545`）
2. 点击 **连接**，程序自动发送通信启动请求（MID0001）
3. 订阅所需的数据类型（拧紧结果、报警、模式等）
4. 通过左侧控制面板进行操作

## 支持的 Open Protocol MID

| MID | 功能 |
|-----|------|
| 0001/0002 | 通信启动/确认 |
| 0003 | 停止通信 |
| 0004/0005 | 命令错误/接受 |
| 0018 | 选择参数集 |
| 0019/0020 | 设定批次/批次清零 |
| 0040/0041 | 请求/接收工具数据 |
| 0042/0043 | 禁用/使能工具 |
| 0050/0053 | 发送/取消 VIN |
| 0060~0066 | 拧紧结果订阅/数据/确认/历史查询 |
| 0070~0078 | 报警订阅/通知/确认 |
| 0224 | 开始拧紧 |
| 0400~0403 | 自动/手动模式订阅 |
| 9999 | 心跳保活 |
